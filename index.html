Mistério de Assassinato em SQL

Você consegue descobrir quem foi o culpado?

Houve um assassinato na Cidade SQL! O "Mistério de Assassinato em SQL" foi projetado para ser tanto uma lição autodirigida para aprender conceitos e comandos de SQL quanto um jogo divertido para usuários experientes de SQL resolverem um crime intrigante.

Novo em SQL?

Este exercício serve mais como uma forma de praticar habilidades em SQL do que um tutorial completo. Se você nunca usou SQL, experimente o passo a passo. Se você realmente quer aprender muito sobre SQL, pode preferir um tutorial completo como o Select Star SQL.

Se você já está confortável com SQL, pode começar abaixo!

Detetives de SQL experientes começam aqui

Um crime ocorreu e o detetive precisa da sua ajuda. O detetive lhe deu o relatório da cena do crime, mas você o perdeu. Você lembra vagamente que o crime foi um assassinato que ocorreu por volta de 15 de janeiro de 2018 e que aconteceu na Cidade SQL. Comece recuperando o relatório da cena do crime correspondente do banco de dados do departamento de polícia.

Explorando a Estrutura do Banco de Dados

Usuários experientes de SQL podem usar consultas para inferir a estrutura de um banco de dados. O Mistério de Assassinato em SQL é construído usando SQLite. Use este comando para encontrar as tabelas no banco de dados.

Além de saber os nomes das tabelas, você precisa saber como cada uma está estruturada. Veja como fazer isso com o SQLite.
